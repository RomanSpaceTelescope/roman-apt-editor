# roman-apt-editor

Helpers for generating long Roman APT programs from TPT review CSVs.

The Astronomer's Proposal Tool (APT) GUI works one observation at a time, which
is painful when a calibration program needs dozens of `<Observation>` blocks
that differ only in their `<LampState>` text. This repo automates that:

- **`rcs_apt_helper.py`** — formats the per-exposure `<LampState>` lines APT
  expects (`<idx>, R=<nres>, LED<N>=<flux> (pre=<dur>,<flux>)`) directly from
  a TPT-style review CSV. Usable as a CLI for ad-hoc snippets, and as a
  library by the populator.
- **`populate_apt.py`** — takes a hand-authored APT seed XML containing one
  `<PassPlan>` with two `<Observation>` templates (a "dark" and a "CRNL Direct
  Illumination"), and produces a populated APT file with **one `<PassPlan>`
  per CSV** — each filled with one `<Observation>` per CSV visit and three
  `<SurveyPlanStep>` entries.

The current driver target is **CAR-173** (count-rate dependent nonlinearity,
direct illumination), but the plumbing is generic to any TPT review CSV that
follows the same column schema.

## Input formats

`populate_apt.py` and `rcs_apt_helper.py` both accept CSV, XLSX, XLS, or XLSM
files. For Excel files, the script defaults to a sheet named `in` if present,
otherwise the first sheet (override with `--sheet`). Both tools validate that
all required columns are present and will report a helpful error if any are
missing.

### Column schema

Both tools consume the TPT review format (one row per *activity*, multiple rows
per *visit*):

| Column | Meaning |
|---|---|
| `VISIT_NUMBER` | Visit grouping key. Rows of the same visit are bundled into one `<Observation>`. |
| `NEXP` | Number of exposures contributed by this row. |
| `RESULTANTS_PER_EXPOSURE` | Per-exposure resultant count (`R=`). |
| `MA_TABLE` | MultiAccum table name (`IM_DIAGNOSTIC`, `DROP_1_OF_2`, …). |
| `SRCS_LEDB1` / `SRCS_LEDB2` | LED on each channel (e.g. `LED16`, `LED26`); blank = unlit. |
| `SRCS_LEDB1_FLUX` / `SRCS_LEDB2_FLUX` | Steady-state flux. |
| `SRCS_LEDB1_PRECHARGE_DURATION` / `…_FLUX` | Optional precharge segment, applied once on the first exposure. |
| `WFI_SRCS_LEDB*_CLEANUP` | Cleanup flag — when set, the running exposure counter resets to 0 for the next visit. |

A visit is treated as **dark** iff every row has both `SRCS_LEDB1` and
`SRCS_LEDB2` blank.

## Usage

### Generate the full populated APT

```bash
python populate_apt.py
```

With no flags this auto-globs `Band*_CFA.csv` or `Band*_CFA.xlsx` next to
`CFA_seed.apt`, alphabetically, and writes `CFA_all_bands.apt`. Override
anything you want:

```bash
# Mix CSV and XLSX files
python populate_apt.py --input Band1all_CFA.csv Band6hf_CFA.xlsx --out CFA_subset.apt

# Use a specific Excel file with a non-default sheet
python populate_apt.py --input 260615_sRCS_WFI_flight_tuning_CFA_APT.xlsx \
                       --seed tuning_seed.apt --out tuning.apt

# Specify the sheet name (default: "in", or first sheet)
python populate_apt.py --input data.xlsx --sheet 'Results' --out output.apt
```

#### Band-1 → Band-2 / Band-3 derivation

The Band 2 and Band 3 CFA tables share Band 1's flux structure but use the
band-keyed LEDs (`LED11→LED12/LED13` on channel B1, `LED21→LED22/LED23` on
channel B2). Rather than maintain duplicate CSVs, the populator detects every
`Band1*_CFA.csv` input and emits derived Band 2 and Band 3 PassPlans in
memory next to it — so `Band1all_CFA.csv` produces three PassPlans labelled
`Band 1 ALL`, `Band 2 ALL`, `Band 3 ALL`. See
[`derive_band` and `collect_sources` in populate_apt.py](populate_apt.py).

If real measured CSVs ever appear for Band 2 / Band 3, drop them in as
`Band2*_CFA.csv` / `Band3*_CFA.csv` and skip the derivation in
`collect_sources` for that variant.

### Print LampState lines from a CSV or Excel file (no XML)

```bash
python rcs_apt_helper.py Band6hf_CFA.csv
python rcs_apt_helper.py 260615_sRCS_WFI_flight_tuning_CFA_APT.xlsx
python rcs_apt_helper.py data.xlsx --sheet 'Results'
```

Prints `===== Visit N - <MA_TABLE> =====` headers followed by the per-exposure
APT lines for that visit. Useful for sanity-checking a file before running the
populator, or for hand-pasting into a single observation in APT.

## How the seed works

`CFA_seed.apt` is a minimal APT program (schema v29) carrying:

1. A single `<PassPlan Number="1">` containing two `<Observation>` blocks
   that differ only in their `<SpecialRequirements>/<Calibration>` payload —
   one with `Type = Dark Imaging`, one with `Type = CRNL Direct Illumination`
   plus an example `<LampState>`.
2. A `<SurveyPlan>` with one `<SurveyPlanStep>` referencing PassPlan 1.

`populate_apt.py` lifts these as templates, clears the live elements, then
clones the templates per CSV / per visit:

```
seed PassPlan          ──→ deep-copied per CSV, <Label> and <Number> stamped, observations replaced
  ├── Dark template    ──→ used verbatim for dark visits, only NEXP / MA_TABLE / Resultant overwritten
  └── CRNL template    ──→ <LampState> filled from rcs_apt_helper.lampstate_for_visit
seed SurveyPlanStep    ──→ deep-copied 3× per PassPlan with fresh 8-char hex uids
```

`<NumberOfExposures>` always equals the *sum* of `NEXP` across the visit's
rows; `<MultiAccumTable>` and `<Resultant>` come from the visit's first row
(per-row `R=` is already encoded inside each `<LampState>` line, so APT's
top-level `<Resultant>` is a default).

To edit the templates' shape (add a new field, change Dither, etc.) edit
`CFA_seed.apt` directly — the populator preserves whatever child elements
you put there.

## Development

The pipeline is small and unit-test-free; verification has been by parity
diff and structural grep. After any change to the formatting logic in
`rcs_apt_helper.py`, sanity-check that the CLI output is unchanged for a
representative CSV:

```bash
python rcs_apt_helper.py Band6hf_CFA.csv > /tmp/after.txt
diff /tmp/before.txt /tmp/after.txt   # must be empty
```

After changing `populate_apt.py`, validate the output:

```bash
python populate_apt.py
xmllint --noout CFA_all_bands.apt
grep -c '<PassPlan Number=' CFA_all_bands.apt          # expect 9
grep -c '<SurveyPlanStep '  CFA_all_bands.apt          # expect 27
```

The ultimate test is to open `CFA_all_bands.apt` in APT itself and confirm
the program loads without schema warnings.

## Authoring notes

- The default XML namespace `http://www.stsci.edu/Roman/APT` is preserved
  on output via `ET.register_namespace('', NS)`; element queries inside the
  populator must therefore namespace-qualify every tag (see the `q()` helper).
- APT auto-snapshots `*.apt` files to `*.aptbackup` whenever it opens them;
  these are already gitignored.
- `Pitch-raster5_v2.csv` and `apt_editor.ipynb` are unrelated to the CFA
  pipeline — they belong to a separate stray-light effort. Left in the repo
  for now; safe to remove if/when that work moves elsewhere.
